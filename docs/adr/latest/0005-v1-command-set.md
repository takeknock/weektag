# 0005. v1 コマンドセット（スコープ確定）

## ステータス
Accepted

## コンテキスト
v1 は PyPI 公開までの距離を最短にしつつ、「記録して、間違いを直せる」が完結している必要がある（0002）。TimeTagger 由来の目標・フォーカスタイマー・タイムライン UI は独立機能であり、後付けしても設計が歪まない。

## 決定
v1 のコマンドは以下：

```
tt start <tags...> [-m メモ] [--at 9:00]   # 開始（実行中があれば自動stop）
tt stop [--at 10:30]                        # 停止
tt status                                   # 実行中タスクと経過時間
tt resume                                   # 直前タスクを同タグ・同メモで再開
tt add 9:00-10:30 <tags...> [-m メモ]       # 後追いの区間追加
tt edit <id前方一致> [--start --stop --tags -m]
tt rm <id前方一致>                          # 削除
tt cancel                                   # 実行中を記録せず破棄
tt log [--week 2026-W27]                    # id付き生ログ（editの入口）
tt report [--week 2026-W27 | --last]        # タグ別集計（0006）
tt export [...]                             # 行TSV/CSV・日報プリセット（0006, 0007）
```

- 目標（targets）・フォーカスタイマー・タイムライン表示 UI・計画（plan）機能は v1 に**含めない**

## 結果
- 打ち忘れ・止め忘れ・誤記録のすべてに回復手段がある
- resume により昼休み明け等の再開が1コマンドで済む（費用対効果が突出して高いため v1 入り）
- v2 候補：目標、フォーカスタイマー、ローカル Web UI タイムライン、plan
