# 0003. 週別JSONLファイルを唯一の真実とする

## ステータス
Accepted

## コンテキスト
確定要件：①Claude Code 等がデータを「そのまま」読める、②振り返りの単位は1週間、③ログを CSV に整形して既存の Excel 時間管理へ転記できる、④バックアップ・移行が容易。SQLite は堅牢だが sqlite3 経由でしか読めず①に反する。個人の時間記録は高々年1〜2万行・数MBで、全読み集計で十分な規模。

## 決定
- ストレージは **JSONL（1行=1レコード）**
- 置き場所は `$XDG_DATA_HOME` 準拠で既定 `~/.local/share/weektag/events/`。環境変数 `WEEKTAG_DATA_DIR` で上書き可。全OS共通のこの1ルールのみとする
- ファイルは **ISO 8601 週番号**で分割：`2026-W27.jsonl`。週の開始は**月曜固定**
- レコードは「**開始時刻が属する週**」のファイルに帰属。週を跨ぐ区間もレコードは分割しない
- 書き込みは一時ファイル→アトミック rename（`os.replace`）
- **週ファイルが唯一の真実**。隠れた状態ファイル・インデックス・DB は持たない。stop / edit はファイルの書き換えで行う。ユーザーやエージェントによる手編集を正式サポートとする
- 実行中タスクの探索は、直近数週のファイル走査で行う

## 結果
- cat / grep / jq / エージェントが前処理なしで読める。「今週分」はファイル名だけで特定できる
- ISO 週年は暦年とズレる（例：2027-01-01 のレコードは `2026-W53.jsonl` に入る）。コマンド側で正しく解決するため実害はないが、手でファイルを探す際の注意点として README に記す
- 週の開始曜日を後から変えると過去ファイルの再分割が必要になるため、**月曜固定は変更不可の前提**とする
- 排他制御は rename の原子性のみ。単一ユーザー CLI の前提で、マルチプロセス同時書き込みは保証しない
