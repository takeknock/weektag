# 0007. 日報プリセット `tt export --daily`

## ステータス
Accepted

## コンテキスト
ユーザーの転記先 Excel は日報型：1行=その日の1タスク、実績は午前/午後に分割。列順はユーザー側で「概要 / 午前実績 / 午後実績 / 予定」に変更済み（**予定を右端**に移動。貼り付けで手入力の予定が上書きされないようにするため）。予定は Excel 側で手入力され、weektag は関与しない。

## 決定
- `tt export --daily [--date 7/6]`（既定は今日）は次の**3列のみ**を TSV 出力する：**概要 / 午前実績 / 午後実績**。予定列は出力しない
- ヘッダー行は**既定で出力しない**（既存の表へ日々貼る運用のため）。`--header` で付与可
- 行の単位＝その日の「**タグ集合＋メモ**」ごとに集約（同一作業の朝夕分割は1行に合算）
- 概要＝`note` があれば note、なければタグを空白区切りで出力
- 午前/午後は**正午 12:00 で機械分割**。跨ぐイベントは按分。境界は `--noon 13:00` で変更可。日跨ぎイベントは当該日の分のみ計上
- 時間は十進・小数2桁丸め。`--round 0.25` で刻み丸め可（既定は丸めなし）

## 結果
- Excel への転記が「3列コピペ」で完結し、右端の予定列は物理的に無傷
- タグ＋メモ単位の集約により、このプリセット内では二重計上が発生せず、日の縦和が実時間と一致する
- 計画機能は weektag に持たせない。必要が生じたら v2 で `tt plan` を検討する
