# 0006. report（集計）と export（転記）の役割分離

## ステータス
Accepted

## コンテキスト
「集計を眺める」と「Excel へ転記する」は出力要件が正反対（整形された表 vs 素の行データ）。1コマンドに混ぜると両方が中途半端になる。ユーザーの既存ワークフローは Excel での時間管理で、時間形式は十進（1.5 形式）を使用している。

## 決定
- **`tt report`**：ターミナル用のタグ別集計表。既定は今週、`--week 2026-W26` / `--last` を指定可
  - 複数タグのイベントは**各タグにイベント全時間を計上**する（タグ縦和は合計を超えうる）。合計行はイベント実時間で別計算する
- **`tt export`**：行レベルのデータ出力
  - 既定は **TSV**（Excel 貼り付けで列が自動で割れる）。`--format csv`、`-o file`、`--no-header` を用意
  - 列は `date / start / stop / hours / tags / note`
- 時間表記は**十進**（例 1.50）。丸めは行わない（`--round` 指定時のみ、0007 参照）
- クリップボード連携は内蔵しない。パイプ（pbcopy / clip.exe / xclip）に委ねる

## 結果
- Excel 側で SUM がそのまま効く
- OS 依存コードが v1 に入らない
- report / export とも出力は**プレーンテキストのみ**。rich の装飾は使わない（パイプ・エージェント・Excel すべてに安全。0009 と整合）
