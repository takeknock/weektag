# 0002. start/stop型＋後追い修正、同時実行は常に1本

## ステータス
Accepted

## コンテキスト
実運用では打ち忘れ・止め忘れが必ず起きる。TimeTagger の価値の核心は「ギャップが見えて、後から埋められる」ことだった。修正手段のないタイマーは数日で使われなくなる。また複数タイマーの同時進行は集計とタイムライン表現を一気に複雑化させる。

## 決定
- 主軸は start/stop タイマー型
- 後追いの add（区間追加）・edit（修正）・rm（削除）・cancel（実行中の破棄）を必須機能とする
- 同時に実行中のタスクは**常に最大1本**。実行中に start されたら、前のタスクを自動 stop してから新タスクを開始する
- データ上は「開始・終了を持つ区間レコード」の1種類に統一する。実行中タスク＝終了時刻を持たないレコード

## 結果
- レコード型が1つで閉じ、別途の状態管理が不要になる
- 並行作業（会議中の内職など）は表現できない。必要が生じたら v2 以降で再検討
- stop / status は「stop の無いレコード」を週ファイルから探すことで実現する（0003 参照）
