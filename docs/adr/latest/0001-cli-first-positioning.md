# 0001. CLIファーストのタグ式タイムトラッカーとして作る

## ステータス
Accepted

## コンテキスト
出発点は TimeTagger の体験——タグだけの低負荷な記録、ギャップが見えて後から直せるタイムライン、単一ファイルによるデータ所有。ただし TimeTagger 自体が既に Python 製・PyPI 公開済みの Web UI ファーストなアプリであり、同等品を作る意義は薄い。本プロジェクトの要件は「Python 製で PyPI 配布、どんな環境でもインストール可能」。

## 決定
- Web UI ではなく **CLI ファースト**で作る。主対象はターミナル常駐の開発者
- 差別化軸は**エージェント可読性**：Claude Code 等のコーディングエージェントが記録ファイルを前処理なしで直接読み、集計・振り返りを手伝えること
- TimeTagger の思想（フラットなタグから後で構造が生まれる・データ所有・シンプルなストレージ）は踏襲する
- 自動トラッキング（ActivityWatch 系のアクティブウィンドウ監視）はスコープ外。OS 依存コードを持たない

## 結果
- pipx / uv での配布と相性が良く、「どんな環境でも」を最も自然に満たせる
- Web タイムライン UI・目標・フォーカスタイマーは v1 に含めない（0005 参照）
- 以後のすべての形式判断は「人間とエージェントの両方がそのまま読めるか」を基準にする
